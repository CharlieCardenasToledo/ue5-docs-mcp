# Contributing to UE5 Docs

Thank you for your interest in contributing. This project has two main parts:

1. **Tooling** — the Python scraper, builder, and MCP server (`scripts/`, `mcp_server/`)
2. **Documentation corpus** — the Markdown files in `docs-md-py/` (auto-generated, not manually edited)

---

## What to contribute

Good contributions include:

- Bug fixes in `scrape_epic_docs.py` or `build_guide.py`
- Improvements to the MCP server (`mcp_server/server.py`)
- New output modes in `build_guide.py`
- Better search logic in the MCP server
- Documentation improvements (README, LLM_INTEGRATION.md)
- GitHub Actions workflow improvements

**Do not** open PRs that manually edit files inside `docs-md-py/`. That directory is generated by the scraper.

---

## Setup

```bash
git clone https://github.com/chcardenasto/DocUnrealEngine.git
cd DocUnrealEngine
pip install -r requirements.txt
pip install -e ".[mcp]"  # if working on the MCP server
```

---

## Workflow

1. **Open an issue first** for any non-trivial change so we can discuss it before you invest time coding.
2. Fork the repo and create a feature branch: `git checkout -b feat/my-improvement`
3. Make your changes — keep commits small and focused.
4. Test your changes:
   - For the scraper: run a small test scrape (`--workers 1`, limit pages)
   - For the builder: run each mode (`--mode index`, `--mode book`, `--mode mkdocs`)
   - For the MCP server: test with `python -m mcp_server.server` and Claude Code
5. Open a pull request with a clear description of what changed and why.

---

## Code style

- Python 3.10+
- Keep functions short and focused
- Use `pathlib.Path` for file paths
- Prefer explicit `encoding="utf-8"` on all file operations
- No external dependencies beyond what's in `requirements.txt` (or `pyproject.toml` extras)

---

## Reporting issues

If the scraper fails on a page or produces malformed Markdown:

1. Check `docs-md-py/FAILURES.md` — the page may already be known to fail
2. Open an issue with the URL and the error message from the scraper output
3. Include your Python version and OS
